# لوحة تجميع بيانات الأبحاث الميدانية

تطبيق ويب (يعمل بالكامل في المتصفح) لتجميع بيانات الأماكن أثناء النزول الميداني،
مع تصنيف الأماكن إلى أقسام (مطاعم وكافيهات، شركات، ... وأقسام مخصّصة)،
وتحديد الموقع تلقائيًا على خرائط جوجل، وتسجيل صوتي يتحوّل إلى نص،
وتصدير تقرير موجّه لجهة معيّنة بصيغة PDF أو Excel.

## المزايا

- **أقسام قابلة للتخصيص**: مطاعم وكافيهات، شركات، وإضافة أقسام جديدة.
- **بيانات كل مكان**: اسم المكان، العنوان، اسم المدير، رقم الجوال، نوع النشاط (15 نوعًا + "أخرى").
- **الموقع على الخريطة**: زر «تحديد موقعي» يجلب إحداثيات GPS، يعرض خريطة جوجل الفعلية،
  ويملأ العنوان تلقائيًا (عبر خدمة OpenStreetMap Nominatim المجانية)، مع خانة ملاحظات يدوية.
- **المقابلات**: تحديد إن تمت المقابلة، وكتابة ما حدث فيها، مع **تسجيل صوتي يتحوّل لنص** تلقائيًا
  (Web Speech API — يعمل على Chrome / Edge).
- **التصدير**: تقرير PDF منسّق أو ملف Excel، موجّه لاسم شركة محدّد ومع اسم الباحث.
- **حفظ محلي**: تُحفظ البيانات في `localStorage` داخل المتصفح.

## التشغيل محليًا

```bash
npm install
npm run dev
```

ثم افتح `http://localhost:5173`.

## الأوامر

| الأمر | الوظيفة |
| --- | --- |
| `npm run dev` | تشغيل خادم التطوير |
| `npm run build` | بناء نسخة الإنتاج في مجلد `dist` |
| `npm run preview` | معاينة نسخة الإنتاج محليًا |
| `npm run lint` | فحص جودة الكود (ESLint) |
| `npm run typecheck` | فحص أنواع TypeScript |

## النشر على Cloudflare Pages

هذا التطبيق ثابت (Static SPA) وجاهز للنشر على **Cloudflare Pages**:

1. اربط مستودع GitHub هذا بـ Cloudflare Pages.
2. اضبط الإعدادات:
   - **Framework preset**: `Vite` (أو None)
   - **Build command**: `npm run build`
   - **Build output directory**: `dist`
3. انشر. ملف `public/_redirects` مضمّن لدعم توجيه SPA.

## ملاحظات

- التسجيل الصوتي وتحديد الموقع يتطلبان اتصال HTTPS (متوفّر تلقائيًا على Cloudflare Pages)
  والإذن من المتصفح.
- خرائط جوجل مضمّنة عبر رابط التضمين العام بدون الحاجة لمفتاح API.
